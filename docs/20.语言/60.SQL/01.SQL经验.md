## 优化

### `group by` 慢查询优化

当表的数据达到千万级别的时候单表查询时携带了 `group by` 会导致一个查询达到 `20s` 。分表这个事情牵扯的太多，只好从语句上进行优化，后面发现了 `DISTINCT` 尝试了一下，直接按到了 `300ms`

``` sql
-- 原SQL
select * from xingcxb group by username
-- 优化后的SQL
select DISTINCT username from xingcxb
```
